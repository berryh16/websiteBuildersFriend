Flask Application

It is important to always import the Flask module into your Flask projects.
To import Flask, just do the following at the top of all documents you are planning to use with Flask:     from flask import Flask

Route Method

The route() function in Flask is a decorator, this means it tells the application which URL it should call for the associated function.

Example:     app.route(rule, options)

The rule parameter defines the URL binding for the function and the options parameter is a list of parameters to be forwarded to the Ruleobject.

Run Method

This method runs the application on the local devlopment server.

Example:     app.run(host, port, debug, options)

The host parameter is the hostname that the server is listening to. By default it is: 127.0.0.1, this is referred to as the localhost.
Similarly, the port parameter is the port number. By default this is set to: 5000.
The debug parameter by default will always be set to false. We will discuss more on this later.
The options paramter, just means to be forwarded to the underlying Werkzeug server.
NOTE: All parameters listed are optional.

For more information about Flask Application you can visit www.tutorialspoint.com .

Debug Mode in Flask

To enable Debug mode on Flask, you would edit the debug property for the
application object to be True before running or passing the debug parameter to the run() method. Once activated the debugger will track any errors in the application.

The following sytanx is what it would look like implemented:
    app.debug = True
    app.run()
    app.run(debug = True)
